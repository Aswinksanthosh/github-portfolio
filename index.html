n8n auto-scheduler

⏰ Stop micromanaging your calendar — let AI do it.
Daily auto‑scheduling with your custom rules. Works while you sleep.
✨ ADHD‑friendly: Only use what you need, no extra apps, just your regular calendar.

⚡ Automation Flow

📋 Google Sheets → ⚙️ Scheduling → 📅 Calendar → 🔗 Webhooks

📖 How your calendar becomes your personal assistant

Every morning, your all‑day tasks from your calendar are automatically converted into timed events, guided by the rules you've set in Google Sheets. They're placed into free slots – respecting your fixed meetings.

You set the rules: buffers, avoid‑events, time windows – and the scheduler obeys without question.

  • ✨ Recurring patterns like “every weekday at 10am” – just write it once.
  • 🔄 Two‑way sync with Google Sheets – changes in the sheet update the calendar, and vice versa. The same data lives in both places, always in sync.
  • 🔗 Clickable webhook links inside event descriptions – mark tasks as FINISHED, POSTPONE, or COMPRESS without leaving Google Calendar. Pure API magic, no extra tools needed.
💡 Real‑world example: Imagine your afternoon is like a train – if one meeting runs late, everything behind it gets delayed. The Connect your events engine automatically shifts your tasks like a railway dispatcher, but more importantly, it enforces smart dependencies: “Gym must be after Work”, “No gym within 4 hours of sleep”, “Cooking before Eating”. Your schedule flows like a story, not a jumble.

🚀 Main Workflow Components

⏰ Scheduling Node

Why it's important: Automates the tedious task of scheduling, ensuring tasks get done at the right time without conflicts.

Background magic: Every 5 minutes, it scans Google Sheets for all‑day tasks. For each task, it parses the configuration (ideal_time, duration, movable, priority). It then checks existing calendar events to avoid duplicates (same task on same date). Using a smart algorithm, it tries the ideal time first, then searches forward/backward in 15‑min increments until a free slot is found. It respects priorities and constraints like buffers and time windows – all while logging every step for transparency.

🔗 Connect your events (Dependency Engine)

Why it's important: Ensures your events respect real‑world relationships – like “Gym must be after Work” or “No gym within 4 hours of sleep”. It's the brain that keeps your day logical.

Background magic: This node runs periodically to resolve conflicts and enforce dependencies. It analyses all timed events, identifies movable ones, and builds chains. It respects complex rules: buffers (gaps before/after), avoid‑events (never schedule near certain meetings), time windows, and dependencies like after_end, before_start, and custom gaps. The algorithm uses a priority‑based system and logs every shift. Think of it as a railway dispatcher that also ensures trains leave only after passengers have boarded.

🔄 CRUD Node

Why it's important: Keeps everything consistent; manual commands allow you to override automation when needed.

Background magic: This node performs bidirectional sync between Google Sheets and Calendar. It detects changes in either system (by comparing descriptions or sheet rows) and propagates updates. It also processes manual commands typed in calendar descriptions: COMMAND: CREATE adds a new sheet entry, COMMAND: DELETE removes it. It validates time/duration inputs and even shows errors right in the description – so you always know if something's wrong.

🔗 Webhook Handler

Why it's important: One‑click control from inside your calendar – instant actions without switching apps.

Background magic: When you click a link in an event description (FINISH, POSTPONE, COMPRESS), an HTTP POST request is sent to an n8n webhook. The webhook identifies the event ID and action, then calls the Google Calendar API to update the event: marking it done (adds ✅ to title), shifting it by the specified minutes, or reducing its duration. All changes appear in your calendar immediately – no extra tools, no manual editing.

Challenges & Solutions

  • Timezone mismatches: Normalised all event times to a single target timezone (America/Toronto) using toLocaleString with timeZone.
  • Duplicate event creation: Introduced a cleanTaskName() function to strip flags and prefixes before duplicate detection.
  • Complex dependency handling: Built a constraints engine that respects buffer times, avoid‑event rules, time windows, and relationships like "after_end" and "before_start".
  • Manual overrides without extra tools: Embedded webhook links directly in event descriptions – clicking them sends an API request to n8n, updating the calendar instantly.
